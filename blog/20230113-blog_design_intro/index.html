Blog Design Intro

January 12th, 2023

Hello! This is the new blog post page design.

Directory Structure

Note that each blog post lives in its own sub-directory within the "blog" directory. This allows you to group all assets with its blog post HTML file. This shortens pathnames for any assets you will call in the HTML, and makes it easier to manage the assets.

Also, each blog post HTML file should be named "index.html" within its directory. This allows the directory path to become the public link without mentioning the HTML file by name. For instance, this post's URL will be:

https://juangmendoza.com/blog/20230113-blog_design_intro/

I have used the naming scheme YYYYMMDD-{blog_post_title}. If you'd rather simplify the links further, you can instead name your blog post directories in numerical order. With that method, this article's URL could instead be:

https://juangmendoza.com/blog/0/

...and the next would be...

https://juangmendoza.com/blog/1/

Using ordered numbers has the benefit of making your URL's shorter, but also makes them less descriptive. Any method works, so long as each blog post directory has a unique name.

HTML Layout

The HTML of this page is relatively simple. The <body> tag of this page is laid out as follows:

<!-- Blog Post Page <body> tag's HTML Layout --> <body> <header>...</header> <main id="blogpost"> <div class="wrapper"> <article>...</article> <aside>...</aside> </div> </main> <footer>...</footer> </body>

Within the <main id="blogpost"> tag, you will notice a <div class="wrapper"> tag. This tag is used to center the page content on the screen with a maximum width set in the CSS. It also adds white space between each blog post entry. Technically, this tag is optional. However, it makes the layout look polished.

There are two element's that make up your blog posts, an <article> element, where the main content of your blog post will go, and an optional <aside> element for a sidebar. The sidebar lies below the article on mobile screens, and to the right on desktop. Excluding the sidebar just leaves that space empty on desktop screens, with no noticeable effect on mobile.

The <article> element is laid out as follows:

<!-- Blog Post Page <article> tag's HTML Layout --> <article> <section id="breadcrumbs"> <a href="/">Home<a href="/">/ <a href="/">Blog<a href="/blog">/ </section> <section id="overview"> <h1>{Blog Post Title}</h1> <span class="date-text">{Blog Post Date}</span> </section> ... <!-- actual blog post content goes here --> </div> </article>

There are two elements that come before the actual content of your blog posts:

  1. <section id="breadcrumbs"> provides links to the homepage and blog page, giving users a sense of where they are in the website. It can be copy/pasted between all blog posts with no change.
  2. <section id="overview"> provides the blog post title, an <h1> tag, and the blog post date, a <span class="date-text"> tag.

After these two elements, you can use whatever other tags are needed to create your blog post. This is pretty bare bones, so if anything ends up not being accounted for, let me know.

Conclusion

This article discussed the directory and HTML structure of the blog post template. Notice that this article intentionally does not have a sidebar. The next article will discuss the different tags you can use for the blog posts, and will contain the sidebar.
